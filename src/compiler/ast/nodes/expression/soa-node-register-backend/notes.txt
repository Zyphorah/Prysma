ON POURRAIT AUSSI FAIRE SANS PROXY, CA SERAIT MOINS SÉCURITAIRE ET MOINS EXTENSIBLE MAIS PLUS PERFORMANT

CHAQUES NOEUDS GARDENT LEURS MÉTHODES AFIN DE GARDER LE CODE COMPATIBLE. L'UTILISATION DE CRTP SERAIT IMINENTE AFIN DE RENDRE LES

IL Y AURAIT AUSSI LA POSSIBILITÉ DE FAIRE DES PROXY, COMME DIT EN HAUT, MAIS QUI SONT SPÉCIALISÉS POUR CHACUNS DES TYPES DE NOEUDS.

    // des types de noeuds

private:
    std::size_t current_id; // pas thread safe je pense

struct NAMES_TAG {}

struct NODE_REGISTRY {
private:
    sparse_set noms;
    sparse_set types;
    sparse_set children;
    // et tous les champs de tous les types de noeuds

    template<typename SET_TAG> struct mapper;

public:
    template<typename SET_TAG>
    auto& emplace(std::size_t id) { return mapper<SET_TAG>::get(*this).emplace(id); }

    template<typename SET_TAG>
    auto& get(std::size_t id) { return mapper<SET_TAG>::get(*this).get(id); }

    template<typename SET_TAG>
    auto& remove(std::size_t id) { return mapper<SET_TAG>::get(*this).remove(id); }
}
static NODE_REGISTRY global_registry;


template<> struct NODE_REGISTRY::mapper<NAMES_TAG> {
    static sparse_set<>& get(NODE_REGISTRY& reg) { return reg.noms; }
};


struct *_NODE {
public:
    auto& getNomFunction() { return global_registry.get<NAMES_TAG>(NodeId); }

private:
    std::size_t NodeId;

}
